# 의미 체계 검색

시맨틱 검색은 AI를 활용해 쇼핑객이 정확하게 입력하는 단어뿐만 아니라 어떤 의미를 갖는지 파악한다. &quot;해변 결혼식 드레스&quot; 또는 &quot;하루 종일 서 있기 편한 신발&quot;과 같은 쿼리는 카탈로그에서 정확한 문구를 사용하지 않는 경우에도 관련 제품을 반환할 수 있습니다.

[!DNL Live Search]은(는) 키워드 일치와 의미 체계 일치를 하나의 검색 경험에 결합합니다. 상점 첫 화면에서 별도의 키워드 및 의미 체계 모드를 관리하지 않습니다. [!DNL Live Search]은(는) 관리자에서 고급 의미 체계 컨트롤(예: 증폭 또는 유사성 슬라이더)을 제공하지 않습니다. 의미 체계 검색을 활성화하거나 비활성화할 수 있습니다.

## 배포별 지원

의미 체계 검색은 [!DNL Live Search] 관리자의 **설정** 작업 영역에서 관리됩니다(**마케팅** > *SEO 및 검색* > **[!DNL Live Search]**). [!DNL Adobe Commerce as a Cloud Service]은(는) PaaS 판매자와 동일한 [!DNL Live Search] 인터페이스를 사용합니다.

## 이점

- **결과 없는 검색 수 감소** — 구매자는 단어가 카탈로그 텍스트와 정확히 일치하지 않으면 제품을 찾습니다.
- **관련 있는 결과 더 보기** — 자연어, 설명 쿼리는 의미와 컨텍스트를 기반으로 유용한 일치 항목을 반환합니다.
- **더 적은 동의어 유지 관리** — 일반적인 단어 변형(예: 소파와 소파)은 종종 수동 동의어 목록 없이 처리됩니다.
- **상점 또는 개발자 작업 없음** — 의미 체계 검색에는 테마 코드, 위젯 또는 API 변경이 필요하지 않습니다. PaaS 판매자는 설정에서 이를 활성화합니다. [!DNL Adobe Commerce as a Cloud Service] 고객은 기본적으로 이를 받습니다.

## 작동 방식

의미 체계 검색이 활성화된 경우 [!DNL Live Search]은(는) 시스템에서 선택한 사전 정의된 카탈로그 특성(예: 제품 이름 및 설명)을 사용하여 기존의 키워드 검색과 함께 쿼리 의미를 해석합니다. 관리자에서 속성을 선택하거나 우선 순위를 지정하지 않습니다.

For example:

- &#39;가죽 소파&#39;를 검색하면 &#39;가죽 소파&#39;라는 라벨이 붙은 제품들이 반환될 수 있다.
- &#39;스프링드레스&#39;는 상품명에 &#39;스프링&#39;이 없어도 계절에 맞는 드레스를 연출할 수 있다.
- &quot;트레일 러닝용 신발&quot;은 오프로드 또는 하이킹 신발로 설명된 제품과 일치할 수 있습니다.

## 의미 체계 검색을 활성화하면 어떻게 됩니까?

의미 체계 검색은 기존 [!DNL Live Search] 구성과 함께 작동합니다. 키워드 검색을 바꾸거나 스토어프론트를 다시 구성하지 않습니다.

의미 체계 검색이 활성화된 경우:

- 기존 [검색 규칙](rules.md), [동의어](synonyms.md), [패싯](facets.md), 부스트 및 [카테고리 머천다이징](category-merch.md) 설정이 계속 적용됩니다.
- 시맨틱 검색은 AI 기반의 쇼퍼 의도 이해를 더해 키워드 매칭과 함께 결과 관련성을 향상시킵니다.
- 사전 정의된 카탈로그 속성은 자동으로 색인화됩니다. 속성을 선택하거나 별도의 구성을 게시하지 않습니다.

## 관리자의 의미 체계 검색 관리

**[!UICONTROL Semantic search]** 토글을 보거나 변경하려면 [설정](settings.md#semantic-search) 작업 영역으로 이동하십시오.

>[!NOTE]
>
> 의미 체계 검색은 **영어** 카탈로그에서만 사용할 수 있습니다. **설정** 작업 영역에서 **언어**&#x200B;를 영어가 아닌 카탈로그로 변경하면 **[!UICONTROL Semantic search]**&#x200B;이(가) 자동으로 비활성화됩니다.

### PaaS 판매자용

Adobe Commerce on Cloud 및 온-프레미스 판매자는 의미 체계 검색을 수동으로 활성화해야 합니다.

1. 관리자의 **마케팅** > *SEO 및 검색* > **[!DNL Live Search]**(으)로 이동합니다.
1. **설정** 작업 영역에서 **[!UICONTROL Semantic search]**&#x200B;을(를) 사용하도록 설정하십시오.

   활성화되면 검색은 의미와 컨텍스트를 기반으로 제품과 일치하므로 더 연관성 있는 결과를 생성하고, 더 적은 수의 결과 검색을 수행하고, 향상된 전환을 수행할 수 있습니다.

1. **[!UICONTROL Save]**&#x200B;을(를) 클릭합니다.

   색인화가 완료된 후 검색 결과가 업데이트됩니다. 중간 크기의 카탈로그의 경우 색인화는 최대 30분이 소요될 수 있습니다. 수백만 개의 제품이 포함된 대규모 카탈로그의 경우 몇 시간이 걸릴 수 있습니다.

### ACCS 고객용

[!DNL Adobe Commerce as a Cloud Service] 고객이 [!DNL Live Search] 관리에서 동일한 **설정** 작업 영역을 사용합니다. 적절한 영어 카탈로그에 대해 의미 체계 검색이 **기본적으로 활성화됨**&#x200B;입니다. **[!UICONTROL Semantic search]**&#x200B;이(가) 활성화되어 있는지 확인하거나, 상점 앞에서 의미 체계를 일치시키지 않으려면 비활성화하십시오.

변경 사항을 저장한 후 별도의 게시 단계 또는 Storefront 구성이 필요하지 않습니다.

## 활성화 후 유효성 검사

시맨틱 검색이 활성화되고 색인화가 완료되면 Adobe에서 검색 성능의 유효성을 검사하는 것이 좋습니다. [성능](performance.md) 작업 영역을 사용하여 지표를 검토하고 비즈니스에 중요한 쿼리를 테스트하십시오. 시맨틱 검색이 기본적으로 활성화되었는지 또는 수동으로 활성화했는지 여부에 따라 적용됩니다.

1. **고유 검색** 보고서에서 가장 많이 검색된 용어를 검토하십시오.
1. 상점 첫 화면의 **0개 결과** 보고서에서 기록된 0개 결과 쿼리를 테스트합니다.
1. 활성화 전후의 동일한 쿼리에 대한 검색 결과를 비교합니다.
1. 클릭스루 비율, 전환율 및 결과 0을 포함한 검색 전환 및 참여 지표를 모니터링합니다.

## 우수 사례

- 명확하고 설명적인 제품 이름과 설명(이상적으로 50~100단어)을 사용하여 키워드와 의미 체계 일치 모두 작업할 강력한 카탈로그 텍스트가 있습니다.
- 의미 체계 검색에 특수 용어가 포함되지 않을 수 있는 브랜드별 또는 고도의 기술적 [동의어](synonyms.md)를 유지하십시오.

## 문제 해결

| 문제 | 할 일 |
| --- | --- |
| 저장 직후 상점 변경 사항 없음 | 인덱싱이 완료될 때까지 기다립니다. 큰 카탈로그는 시간이 더 걸릴 수 있습니다. |
| 의미 체계 검색을 사용할 수 없거나 자동으로 비활성화됩니다. | **설정** 작업 영역의 **언어**&#x200B;이(가) **영어**(으)로 설정되어 있는지 확인하십시오. |
| 결과가 여전히 공통 용어를 누락함 | 브랜드 또는 업계 용어에 대한 [동의어](synonyms.md)를 추가해도 의미 체계 검색이 해결되지 않을 수 있습니다. |

## 제한 사항 {#semantic-search-limitations}

- **카탈로그 언어:** 의미 체계 검색은 **영어** 언어 카탈로그에서만 사용할 수 있습니다.
- **관리자 컨트롤:** [!DNL Live Search]에서는 사용/사용 안 함 컨트롤만 제공합니다. **설정** 작업 영역에서 의미 체계 증폭, 유사성 임계값 또는 유사 항목 검색을 조정할 수 없습니다.
